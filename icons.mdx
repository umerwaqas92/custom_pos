# POS Icons Style Guide & Prompts

This document lists all custom icons used in the app, where they live, the shared generation style, and the exact prompts used so you can regenerate or extend the set.

---

## Style system

Use this style for every new icon so the set stays consistent.

### Visual rules

| Rule | Value |
|------|--------|
| Format | Transparent **PNG** (RGBA) |
| Canvas | Square `1:1` (export ~128–256px for UI) |
| Shape | Flat modern UI symbol, centered |
| Contrast | High contrast; avoid pale/white-only icons |
| Text | No text, no labels, no watermarks |
| Shadow | No drop shadow on chroma sheet (optional soft depth in symbol only) |
| Colors on symbol | Blue / indigo / amber / rose / teal / gold / white / navy — **never pure chroma green** |
| Background for generation | Solid pure chroma key green `#00FF00` (full frame) |
| After generate | Remove green → pure transparent PNG |

### Chroma key prompt suffix (always append)

```
on solid pure chroma key green #00FF00 background only, full frame green, NOT green colors on the icon itself, flat modern UI, high contrast, no text, no drop shadow, square 1:1
```

### Post-process pipeline

1. Generate on solid `#00FF00`.
2. Sample green from image corners as key color.
3. Remove pixels that are greenish **and** close to the key color.
4. Soft-edge despill on fringes.
5. Crop to content + pad, fit square, resize (sidebar `128`, dashboard `256`).
6. Slightly darken near-white pixels if needed for light UI.
7. Save as PNG with alpha.

### App usage paths

```
frontend/public/icons/
  dashboard/     → home KPI widgets
  sidebar/       → main navigation
```

Dashboard cards use soft tint plates (`bg-sky-500/15`, etc.).  
Sidebar items use `bg-primary/10` (inactive) / `bg-white/20` (active).

## Template: new icon

Copy and fill:

```
[ROLE: Sidebar nav icon | Dashboard widget icon]
on solid pure chroma key green #00FF00 background only, full frame green.
Centered [COLORS: white + accent color] [SYMBOL description].
Flat modern UI, high contrast, NOT green colors on the icon itself.
No text, no drop shadow, square 1:1
```

Then:

1. Save generate output as temp JPG.
2. Run chroma → transparent PNG.
3. Place under `frontend/public/icons/dashboard/` or `sidebar/`.
4. Reference in `Dashboard.tsx` or `Layout.tsx` as `/icons/.../name.png`.
5. Add a row to this file.

## Do / Don’t

**Do**

- Always generate on `#00FF00` for easy keying.
- Use saturated accent colors on the symbol (blue, gold, rose, etc.).
- Keep symbols simple and readable at 20–28px.

**Don’t**

- Don’t put green, lime, or emerald fills on the symbol (they disappear in chroma key).
- Don’t add text or multi-scene backgrounds.
- Don’t ship JPG to production UI; use transparent PNG only.
